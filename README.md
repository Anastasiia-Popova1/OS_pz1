# Пояснення по завданням

## Завдання 1

При виконанні цього завдання були використані основні команди вводу та виводу cin та cout, а також бібліотека iomanip, яка відповідає за форматування вивіду.

## Завдання 2.1

При виконанні цього завдання була викоритсана бібліотека cmath, яка запеспечила тригонометричні та звичайні обчислення. Основний алгоритм рішення зав'язан на операторі вибору if, який порівнює значення z з умовами z > 0, 0 <= z <= 8, z > 0. В залежності від виконаної умови обирається функція, за допомогою якої обчислюється кінцевий результат.

## Завдання 2.2

При виконанні цього завдання буd використаний оператор switch, саме він виконує основну логіку програми: перехоплює цілочисельне значення n, яке відповідає за одиницю виміру, а потім порівнює зі значеннями, які йдуть після case. Якщо значеня було піймано, то друкує довжину відрізка в метрах, яка обчислюється. Оператор break запеспечує своєчасний вихід. Якщо було піймано число поза умовою, друкується "Помилка".

## Завдання 3.1

При виконанні цього завдання був використаний цикл for, який відповідає за почергове введення чисел та пошук мінімального й максимального елемента. Він виконується n-1 кількість разів, так як перше число вводиться окремо, поза циклом, щоб ініціалізувати змінні min та max. Далі в циклі кожне наступне число порівнюється з поточними мінімумом та максимумом, і за потреби змінні оновлюються. Після завершення циклу обчислюється різниця max - min та виводиться результат.

## Завдання 3.2

При виконанні цього завдання були використані цикл while та цикл do-while для обчислення функції F(x) = √х · cos²х на заданому інтервалі. Обидва цикли демонструють почергове обчислення x та відповідної функції F(x) з кроком h. Для виведення результатів у вигляді таблиці використовуються бібліотека iomanip, а саме маніпулятори fixed, щоб коректно відобразити дробове число, а також setprecision() та setw(), які вже раніше були викоритсані для відображення фіксованої кількості цифр після коми та для визначення мінімальнох ширини поля. Для обчислення ми кикористовували бібліотеку cmath. Було враховано, що підкореневий вираз не може бути від'ємним, при x < 0 виводиться "не визнач.". 
Відмінність цих циклів в тому, що в while спочатку перевіряється умова (while (x <= b)), потім тіло циклу. В do-while прийшлося додати перевірку if (x <= b) з самого початку, так як основна умова циклу знаходиться в самому кінці (while (x <= b)).
